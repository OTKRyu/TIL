# tdd

- 테스트 기반 개발
  - 코딩은 결정, 피드백의 연속
  - 피드백 받을 테스트 케이스 생성
  - 코드 먼저 테스트 나중이 아닌 테스트케이스 먼저 코드 나중
  - 테스트를 어렵게 만드는 것
    - 불확실성: 임의의 값이 필요한 경우, 전역변수, api서버 등
    - 부수작업: db에 기록, 메일 발송, 시스템 외부에 뭔가를 하는데 리턴은 없는 것들
- pytest
  - python의 테스트를 도와주는 라이브러리
- 장점
  - 디자인 패턴을 의식하지 않고 어느 정도 이에 따른 코드 결과물을 만들 수 있다
  - 테스트 코드 고민이 줄어든다
- 단점
  - 테스트 케이스의 퀄리티에 따라 결과물이 달라진다
  - 발사의 전환에 시간이 걸린다
